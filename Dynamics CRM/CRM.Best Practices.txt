5) QueryExpression для Equal не зависит от регистра, то есть эти 2 запроса будут одинаковыми

var query = new QueryExpression("real_estate_object");
query.ColumnSet.AddColumns("code");
query.Criteria.AddCondition("code", ConditionOperator.Equal, "алх1/1к01-1-02-01-001");

var query = new QueryExpression("real_estate_object");
query.ColumnSet.AddColumns("code");
query.Criteria.AddCondition("code", ConditionOperator.Equal, "АЛХ1/1К01-1-02-01-001");


(!) Но в crm 2011 зависимость от регистра есть

6) Плагин на удаление карточки

Если шаг плагина зарегистрирован на удаление, то здесь
context.InputParameters["Target"]
будет объект типа EntityReference, а не Entity

Если хочешь получать связанные записи с удаляемой карточкой, то плагин необходимо регистрировать на PreValidation

7) Для Действий нельзя использовать аргумент с именем "Error", "Date" - будет падать ошибка

8) Ошибка при импорте решения на среду #1
	Если при импорте решения на среду возникает подобная ошибка Unable to load the plugin type: ORG.Plugins.Email.SendSystemEmail
	Скорее всего причина в том, что был добавлен новый плагин. Можно попробовать перенести в 2 решения. Сначала переносится сборка. А затем уже шаг обработки. 
	На некоторых средах бывает так. что при импорте CRM сначала пытается обновить шаг обработки, а затем сборку. 
	Но из-за этого, что шаг зависит от сборки, выполнить это не удается 
	
9) Ribbon Workbench
	Если указать несколько экшенов (JS-скриптов) для выполнения, то они выполнятся последовательно
	В случае, если один из экшенов (JS-скриптов) упадёт, последующие все равно выполнятся
	
10) Элемент очереди
	При создании элемента очереди, для его отображения пользователь должен быть участником очереди
	
11) Подгрузка библиотеки для кнопки
	Если в скрипте для кнопки на гриде или в представлении требуется какая-то дополнительная библиотека, то можно в enable rule добавить правило, 
	в котором мы подгружаем это библиотеку, а в качестве вызваемой функции используем isNaN JS.

16) Не отображается изображение на карточке у некоторых пользователях.
	Если создано несколько организаций и пользователь заведён также в нескольких, то происходит ошибка отображения, т.к. url указывается без имени организации
	(если к ссылке добавить имя организации, то изображение отобразится нормально). Если пользователь будет заведён только в одной организации, то всё будет норм.
	
	https://community.dynamics.com/crm/f/microsoft-dynamics-crm-forum/385678/entity-image-broken-on-dynamics-365-v9-on-premise
	
	
17) Не отображается AutoComplete на форме быстрого создания
	Причина в том, что на этой форме отсутствует контрол Поиска (lookup), необходимо добавить любой лукап на форму (например, ответственного). Поле можно сделать скрытым
	
	https://stackoverflow.com/questions/35469720/crm-2016-autocomplete
	https://community.dynamics.com/crm/f/microsoft-dynamics-crm-forum/223528/auto-complete-for-quick-create-forms-in-2016-365/603530
	
	
18) Порядок выполнения js функций на форме
	(!) Порядок важен!!!
	
	Если на форме задано 2 Скрипта
		Скрипт 1 (Функция 1.1)
		Скрипт 2 (Функция 2.1)
		
	И на загрузку формы они висят в порядке
		Функция 2.1
		Функция 1.1
		
	То сначала будет выполнена функция 2.1, затем только 1.1

21) Ошибка: There is no active transaction. This error is usually caused by custom plug-ins that ignore errors from service calls and continue processing.
